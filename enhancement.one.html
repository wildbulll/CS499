Description of the Artifact:

This artifact is an interactive dashboard web application created using Dash and Python. It was initially developed in March of 2025 as part of the CS-340 course to display and analyze animal rescue data through tables, charts, and maps. The original version had basic layout and filtering options.

Justification for Inclusion in ePortfolio:

This dashboard shows that I can build a complete data visualization tool from start to finish while managing both the design and functionality of a full-stack application. I created an interactive layout that includes a data table, pie chart, and map, with each component connected to live data and organized using a responsive Flexbox structure so they adjust when the screen is resized. The dashboard pulls live data and updates the charts and map automatically, which shows I can connect data to visual displays. I also customized the colors, layout, and spacing to make the app easy to use and visually appealing.

There are still a few options that need improvement, which I plan to address in the next milestone when I work on the database.

Outcomes Achieved in This Milestone:

  • Better Layout with Flexbox: Previously, the dashboard stacked everything vertically with no clear order. I used Flexbox to create side-by-side columns, making it easier to view and use.
  • Improved Visual Design: I added a round logo, bold title, and consistent visual style with colors, spacing, and shadows.
  • Organized and Clear Code: I grouped related elements and added comments to make the code easier to understand and maintain.
  • More User-Friendly Data Table: I added scroll functionality and kept sorting/filtering features to improve data exploration.
  • Consistent Styling: I applied matching styles to dropdowns, buttons, and headings and set fixed sizes for charts and maps.
  • Smarter Data Options: I simplified dropdown choices in the pie chart to reduce clutter and improve usability.

Outcomes and Updates:

These changes helped me meet the goals I set earlier in the course. I focused on creating clean designs and writing maintainable code. I now understand more about responsive web design and how user experience can be improved through visual layout and interaction design. I will continue learning more about accessibility and performance to make my future projects even better.

Reflection on the Enhancement Process:

I improved this project by adjusting layouts to fit different screen sizes, making the app more usable and visually clear. Balancing function with design taught me to prioritize the user experience. I faced challenges arranging components clearly and debugging layout spacing, but small iterative changes helped me resolve issues. I also made the pie chart dropdown smarter by limiting its options to only relevant columns using debugging and code inspection.
